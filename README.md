## 📊 Lighthouse Performance Scores

The **FLEXICAR** project has been analyzed with [Lighthouse](https://developers.google.com/web/tools/lighthouse) for web performance, accessibility, SEO, and best practices.

### Performance Scores:

| Metric             | Score   |
| ------------------ | ------- |
| **Performance**    | 94/100  |
| **Accessibility**  | 91/100  |
| **Best Practices** | 96/100  |
| **SEO**            | 100/100 |

**Key Highlights:**

- The **Performance** score reflects fast load times with optimized assets, such as deferred JavaScript, preloaded styles and lazy-loaded images.
- **Accessibility** is optimized with semantic HTML and proper ARIA roles, ensuring that screen readers can effectively interpret the content.
- **Best Practices** cover security aspects, ensuring HTTPS usage and minimizing vulnerabilities.
- **SEO** ensures that the site is properly indexed by search engines with correct meta tags, sitemaps, and structured data.

## 🌐 SEO & Accessibility

- **Semantic HTML**: The project uses semantic HTML elements to improve structure and readability for both users and search engines.
- **Alt text for images**: All images include descriptive alt text for accessibility.
- **Readable typography**: Clear and legible typography choices are made for better readability.
- **Color contrast meets WCAG standards**: Ensuring sufficient contrast between background and foreground for users with visual impairments.

## 🧰 Running the Project Locally

To run the project locally, follow these steps:

```bash
# 1. Clone the repository
git clone https://github.com/OlgaBieliaieva/flexicar

# 2. Navigate to the project directory
cd flexicar

# 3. Open index.html in your browser
# Or you can open it directly using Live Server in VS Code.

```
